# SKILL 3：PPT生成

## 触发方式
用户说"生成PPT"、"出幻灯片"、"做模块X的PPT"

---

## 技术路线

基于 **next-slide** 方案——Markdown → HTML演示文稿，零依赖，50+风格，Claude Code原生支持。

> ⚠️ **注意**：本SKILL定义的是PPT内容的生成格式和规则。实际转换为PPTX文件或HTML演示文稿的技术步骤，在后续实际制作PPT时再调优。

---

## PPT内容生成流程

### 第一步：确定输入
- 课程模块的完整内容（来自SKILL 2的输出或 `course-output/` 下的模块文件）
- 知识库中的支撑素材
- 目标风格（商务/科技/极简/黑金——默认"商务"）

### 第二步：内容结构——每页PPT的标准格式

```
【PPT页 X/Y】
类型：标题页 | 信息页 | 案例页 | 数据页 | 互动页 | 过渡页
标题：（一句话，不超过15字）
核心内容（3-5 bullet points，每点不超过20字）：
• xxx
• xxx
视觉建议：（描述图表/图片/图标）
演讲备注：（讲师说的话，比屏幕上的字多3-5倍）
素材来源：（引用知识库哪条）
获客暗示：（这一页如何推动信任或获客？如不适用则留空）
```

### 第三步：模块PPT的页面结构模板

```
模块PPT标准结构：

[1/总页数] 标题页
- 模块名称 + 一句话核心承诺

[2] 开场钩子
- 一个故事/数据/问题——让老板坐直

[3] 目标预告
- "接下来X分钟，你会得到什么"

[4-N] 小节1：核心信息页
- 每页一个核心信息
- 配合案例/数据/类比

[N+1] 小节1：支撑素材页
- 展示1-2个具体案例或数据

[N+2-M] 小节2：核心信息 + 支撑

...

[X] 互动页
- 讨论题/自测题
- 简洁到学员10秒内能理解要做什么

[倒数2] 收尾页
- 本模块金句
- 一句话过渡到下一模块

[倒数1] 行动提示页
- 让学员产生"我需要找人聊聊"的念头
```

### 第四步：PPT设计原则

**老板级PPT的铁律**：
1. **每页一个核心信息**——不要在一页上放两个观点
2. **子弹点不超过5条**——每条不超过20字
3. **数据一定要有对比**——"提升了30%"比"提升30%"更有力（写了从多少到多少）
4. **类比比定义更有效**——能用类比解释的概念，不要用定义
5. **每3-4页切换节奏**——信息页→案例页→数据页→互动页，不要让同一类型的页面连续超过3页
6. **演讲备注的字数是屏幕文字的3-5倍**——屏幕给骨架，嘴给血肉

**视觉风格（默认商务风）**：
- 深色背景+白色文字，或白色背景+深蓝/黑文字
- 少图多符号（图标＞照片＞截图）
- 字体大小：标题 36pt+，正文 24pt+
- 每页留白率不低于40%

## next-slide 转换参考（待实际制作时使用）

```
将课程模块内容整理为 Markdown 格式 →
选用 next-slide 风格（推荐：business/dark/tech） →
Claude Code 调用 next-slide 生成 HTML →
浏览器打开预览 →
调整内容/风格 →
导出最终版本
```

**next-slide的Markdown输入格式**：
```markdown
---
theme: business
---

# 模块X：[标题]
## [一句话核心承诺]

---

# [小节标题]
- 核心信息1
- 核心信息2
- 核心信息3

--

# 案例：[企业名称]
## [一句话案例标题]
- 做了什么
- 关键数字
- 对老板的启示

---

# 讨论
## [互动问题]
```
