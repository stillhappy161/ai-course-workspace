# SKILL 1：素材摄入与提炼

## 触发方式
用户说"存一下"、粘贴文章/链接/转录文字、或输入"提炼：[内容]"

---

## 智能体自动执行流程

### 第一步：结构化提取
从原始素材中提取：
- **摘要**（3-5句，用老板能理解的语言）
- **关键数字/引述**（可量化的优先，一句金句次之）
- **核心观点**（1-3条，每条一句话）

### 第二步：自动分类与标签
自动判断并标注：
- **知识库分类**：应该存入哪个文件？（案例/框架/数据/概念/故事/痛点/类比/学习材料）
- **适用模块**：M1认知/M2评估/M3路径/M4投入/M5组织/M6行动
- **素材级别**：
  - ⭐⭐⭐ 核心——必须进课程，不可替代
  - ⭐⭐ 重要——进课程，可被更好的素材替换
  - ⭐ 备选——暂存，视课程发展决定是否使用
- **可用性**：#可入课 / #待验证 / #仅参考

### 第三步：自动生成一个互动问题
基于素材内容，生成一个可以在课程中使用的讨论题或自测题。
格式：`[互动] 问题文本（类型：小组讨论/全场投票/自测/沙盘）`

### 第四步：关联检查
- 这条素材和我们已有的哪条素材有关联？（补充/矛盾/印证）
- 如果发现矛盾观点，标注 `⚡ 观点冲突`
- 如果填补了某个已知缺口，标注 `✅ 缺口填补：[缺口名称]`

### 第五步：写入知识库
按格式写入对应的知识库文件，标注贡献者和日期。

### 第六步：MOC更新提醒
检查是否需要更新以下文件：
- `module-content-packages.md`——新素材是否改变了模块素材映射
- `course-structure.md`——新素材是否触发了课程结构调整

---

## 输出格式

```
📥 素材已入库

📁 存入：[文件名]
🏷️ 分类：[案例/框架/数据/概念/故事/痛点/类比]
📌 适用模块：[M1-M6]
⭐ 级别：[核心/重要/备选]
🔗 关联：[已有素材标题]（补充/矛盾/印证）
💬 互动问题：[生成的讨论题]

❓ 追问：[基于素材的追问，帮助深化理解]
```
