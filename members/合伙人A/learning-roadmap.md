# 学习路线图

## 你的角色

你不是学员——你是课程的**技术底座**。主编（管理咨询背景）负责商业框架，你负责确保课程中每一个技术表述都是准确的、每一个技术方案都是可行的。

你需要做到：
1. 审核全部课程内容的技术准确性——不能让老板听完课回去问CTO，CTO说"那个人讲得不对"
2. 为课程的技术环节提供"老板能听懂的版本"——把技术复杂度翻译成决策语言
3. 当主编提出一个商业想法时，你能给出技术实现的具体路径
4. 为课程设计"技术答疑角"——专门回答老板现场提出的技术问题

---

## 第一阶段：审核现有课程内容的技术准确性（第1-3天）

- [ ] 通读全部知识库文件（26框架+12概念+12案例+10痛点），标注所有技术表述不准确的地方
- [ ] 通读课程六个模块的完整内容（M1-M6），逐页审核技术表述
- [ ] 重点审核：AI能力边界（读写问答找调——技术层面准确吗）、技术栈四层（分层对吗）、RAG/Agent/MCP的解释
- [ ] 为每个技术概念补充一句话的"老板版解释"（如果主编的版本不够准确，给出修正版）

**检验标准**：如果把课程内容拿给另一个AI工程师看，他不会说"这讲得有问题"

---

## 第二阶段：技术路线图验证（第4-5天）

- [ ] 验证AI落地咨询四阶段路线图（IBM+麦肯锡）的技术可行性
- [ ] 对每个B类案例（冰山松洋、联合轴承等）的技术栈做反向分析——他们实际用了什么、怎么实现的
- [ ] 为B类企业设计三条"技术实现参考路线"——最低成本路线、最稳定路线、最灵活路线
- [ ] 验证四个场景（客服/合同/营销/供应链）的技术落地路线在2026年是否仍然适用

**输出**：技术路线验证报告 → 存入 knowledge-base/ 或 members/合伙人A/

---

## 第三阶段：技术选型指南（第6-7天）

- [ ] 基于课程内容，设计一份"B类企业AI技术选型决策表"
  - 什么场景用什么工具（客服→FastGPT or Coze？合同→RAGFlow？）
  - 什么情况下要私有化部署、什么情况下SaaS就够了
  - 不同规模的B类企业（30人 vs 300人 vs 3000人）分别该选什么技术方案
- [ ] 为课程M4投入篇补充"技术成本拆解"——不是笼统的"30-200万"，是按技术栈拆开的真实成本
- [ ] 为M5组织篇补充"IT部门升级路径"——从修电脑到搭AI平台，分几步、多久、花多少钱
