# 学习路线图

## 学习目标

你的目标不是做研究，是**成为一名能独立讲授《从0到1搭建企业AI能力》的讲师**。

你需要做到：
1. 能独立讲清楚课程的每个模块（不需要像主编那样深刻，但必须准确、流畅、能回应学员提问）
2. 能用学员能懂的语言解释AI概念（少用技术术语，多用类比）
3. 能在课堂上应对各种"刁钻"提问（挑剔的老板、怀疑的中层、焦虑的创业者）
4. 能配合主编完成课程交付——主编讲框架和战略，你补充技术细节和落地案例

---

## 学习路径（跟着主编的节奏走）

### 第一阶段：追赶知识库（第1-3天）

你不是从零开始——主编和智能体已经积累了14个框架、9个概念卡、10条痛点、7组数据、6个案例、3个金句。你的第一任务是**把已有的东西全部消化**。

- [ ] 通读全部知识库文件（knowledge-base/ 目录下所有 .md 文件）
- [ ] 通读课程结构草稿（course-output/course-structure.md）
- [ ] 通读主编的学习路线图和提问日志（理解主编的思维路径）
- [ ] 标注你不理解的概念和框架，向智能体提问
- [ ] 用自己的话重新解释每个核心框架（存入你的 question-log.md）

**每天结束时问自己**：我今天读的内容，能不能用5分钟讲给一个完全不懂AI的人听？

---

### 第二阶段：同步学习（第4天起）

主编继续学习新内容时，你同步跟进。每学一个新框架/新概念：

- [ ] 先用自己的理解复述一遍，让智能体评价你的复述是否准确
- [ ] 给每个概念找一个新的类比（不能和主编已有的重复）
- [ ] 想一想：如果课堂上学員问"这跟我有什么关系"，你怎么回答

---

### 第三阶段：讲课练习（持续进行）

每天至少做一次讲课练习。在对话框中输入"讲课练习：[模块名称]"。

智能体会扮演不同类型的学员向你提问：
- **犹豫型**："我觉得AI是趋势，但我们的情况不太一样……"
- **挑剔型**："你说的这个ROI框架，我公司数据都是乱的，怎么算？"
- **技术崇拜型**："是不是我们应该直接上最新的GPT-5？"
- **焦虑型**："你说AI能替代客服，那我客服团队怎么办？"
- **务实型**："说这么多，我明天回公司该先做什么？"

你需要现场回应，然后智能体会评价你的回应质量：准确度、可理解度、说服力。

---

### 第四阶段：协作备课（未来）

- [ ] 为主编提供的课程内容做"学员视角审查"——如果你听不懂，学员更听不懂
- [ ] 为每个模块提供技术落地细节的补充（你比主编更懂技术）
- [ ] 参与课程互动的设计——站在学员角度想"什么样的讨论能让老板真正思考"
- [ ] 准备"技术答疑角"——课程中专门留一段让你解答老板的技术疑问

## 你的优势
