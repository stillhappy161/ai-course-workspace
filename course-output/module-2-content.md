# M2 评估篇：我的企业在哪个起点 | 55min

## A-分析层

- **学员状态**：刚听完M1，知道AI能做什么了。但心里在想"这跟我有什么关系"、"我公司能行吗"
- **核心痛点**：不知道自己处于什么位置、不知道别人在什么位置、不知道自己卡在哪
- **信任建立机会**：用2×2矩阵做"即时的自我诊断"——老板5分钟就能对号入座。这是整个课程"最懂我"的时刻
- **获客触发设计**：诊断出自己是B类→"问题清楚了，但B类的解在哪？"——M3回答。但光知道路径不够，还需要有人帮你走

---

## D-设计层

| 维度 | 内容 |
|---|---|
| **核心问题** | 我的企业现在在哪个起点？我最可能卡在哪一层？ |
| **CEO三层关切** | 战略落地（定位=选择路径的起点）+ 利润表（数据不准的代价）+ 估值（数字化成熟度=资本故事的可信度） |

| **前模块** | M1——"知道了能做什么，现在判断该从哪开始" |
| **后模块** | M3——"知道了起点，选你的路" |

## D-开发层

### Part 1：一张2×2矩阵——5分钟判断你的起点（15min）

**【PPT 1/20】过渡页**
- 标题：刚才你知道了AI能做什么。现在回答一个更关键的问题——

**【PPT 2/20】钩子——"95%的AI项目死在概念验证"**
- 核心信息：MIT：95%的项目没过PoC。麦肯锡：90%启动，10%规模化。为什么？因为大多数人第一步就走错了——没搞清楚自己在哪
- 演讲备注："各位，在AI这件事上，你最大的风险不是技术跟不上，是起点判断错了——A类企业用了C类的做法会浪费钱，C类企业用了A类的做法会浪费更多钱。这55分钟，就是帮你找到正确的位置。"

**【PPT 3/20】2×2矩阵——四起点模型**
- 核心信息：两个维度——数据基础（好/差）× 组织技术能力（强/弱）→ 四个格子：
  - 🟢 A 子弹上膛：数据好+能力强——三一、海尔、伊利
  - 🟡 B 有矿不会挖：数据有但散+能力弱——**大部分中国企业在这里**
  - 🟡 C 两手空空：数据差+能力弱——中小企业
  - 🔴 D 有枪没子弹：数据差+能力强——科技创业、空降CTO
- 演讲备注："我来猜一下你们在哪里。不要说出来，自己在心里打分——两个问题：你公司的核心经营数据，一小时内能拉出来吗？你公司有没有一个人能说清楚'我们用了什么系统、每个系统里有什么数据'？如果两个都是Yes，你在A。两个都是No，你在C。一个有、一个没有——猜猜你在哪？"
- 视觉建议：大号2×2矩阵，四个格子分别标注A/B/C/D

**【PPT 4/20】互动——现场自测**
- 类型：全场举手
- 问题："觉得自己属于B类的，举个手。"
- 引导提示：大量人会举手。讲师："你们知道吗——我在全国讲了这么多场，几乎每次都是B类最多。你们不是落后，你们是正常的。中国的企业，大多数就在这个格子里。"
- 获客设计：让老板意识到"大家都在同一艘船上"→焦虑缓和→但也意识到"这么多人卡在这里，说明这条路不好走"

**【PPT 5/20】A类——子弹上膛的少数派**
- 核心信息：三一、海尔、伊利——系统跑了好几年，有CTO或数字化团队。他们的问题不是"能不能做"，是"先做哪个"。A类企业的课不是今天要讲的
- 演讲备注："如果你在A类，恭喜你。你今天的收获会是对标——看看别人A类做了什么，验证你自己的想法。"

**【PPT 6/20】B类——有矿不会挖的主力军**
- 核心信息：年营收5-30亿，ERP/OA/财务系统上了，但各部门数据不互通。IT部门有，但只做运维。老板知道"数据是资产"，实际上数据躺在"数据坟墓"里。**B类企业的第一步不是上AI，是搞清楚自己有哪些数据、数据准不准**
- 演讲备注："你觉得自己在B类。那我要问你一个更扎心的问题——你的数据，准不准？"(停顿)"不要急着点头。我见过太多B类企业——ERP里的库存和仓库实物对不上、CRM里的客户信息三个月没更新、销售数据和财务数据报表永远对不齐。"

**【PPT 7/20】C类——两手空空的起步者**
- 核心信息：数据在Excel和微信里，没有IT团队。不是落后，是起点不同。C类企业的优势：决策链短，可以直接买SaaS，不需要被数据治理困住
- 演讲备注："C类企业有一个B类企业没有的优势——你们没有'数据坟墓'。你们的数据虽然少，但干净。B类企业的数据虽然多，但脏。脏数据比没数据更危险。"

**【PPT 8/20】D类——有枪没子弹的困局**
- 核心信息：团队强、技术好，但数据基础差。空降CTO、科技创业公司、集团"数字化特区"。他们有最快的车，只是还没铺路
- 演讲备注："D类在座的可能不多。但如果你在——你的困境是'我的人能干活，但没东西让他们干'。解决方案要么是先攒数据，要么是先用外部数据跑起来。后面M3会讲。"

### Part 2：数据质量陷阱——B类企业的隐藏病灶（15min）

**【PPT 9/20】过渡——"你确定你的数据能用吗"**
- 标题：数据不准——比没数据更危险

**【PPT 10/20】三个典型场景**
- 核心信息：
  - ERP库存数据和仓库实物对不上→你基于错误库存做的AI预测，还不如不做
  - CRM客户信息三个月没更新→AI给客户打电话喊错名字
  - 销售的CRM、生产的MES、财务的用友——三个系统从不说话→AI不知道"签了一个大单"意味着生产和财务都要动
- 演讲备注："你们公司有没有这种情况——同一个客户在三个系统里有三个不同的名字？'张三'在CRM里是'张三'，在ERP里是'张总'，在财务系统里是'深圳张'。你认为AI能认出来这是同一个人吗？认不出来。它以为你服务了三个客户。"
- **这是整个M2的获客触发点**：老板突然意识到——"我公司好像就是这样"

**【PPT 11/20】数据不准——已知的未知 vs 未知的未知**
- 核心信息：没数据你知道自己不知道（已知的未知）。数据不准你以为知道但其实不知道（未知的未知）。后者危险十倍。很多B类企业上AI的第一步不是买模型，是花3个月搞清楚"我到底有什么数据、哪些能信"
- 演讲备注："你有没有遇到过这种情况——汇报会上，销售总监说上个月营收5000万，财务总监说不对，是4800万。两个人对不上，然后花半天找到原因——一个算含税、一个算不含税。这种数据，你敢喂给AI吗？"

**【PPT 12/20】翻车故事——Builder.ai**
- 内容：13亿美元的"假AI"。供应商跟你说"AI能做"，实际上是人肉
- 演讲备注："怎么判断供应商是不是真的AI？问他：'它能处理没见过的情况吗？'答不出来的——不是AI。数据不准的情况下，AI的判断还不如一个有经验的员工。"

### Part 3：四层就绪度速查——你卡在哪一层（15min）

**【PPT 13/20】过渡——"厨房诊断法"**
- 标题：外面超市什么都有。你家厨房能不能做饭？

**【PPT 14/20】四层就绪度模型**
- 核心信息：从上往下走——先想清楚做什么菜、再看料够不够、再看灶行不行、最后看有没有人吃
  - **业务层（菜谱）**：你业务中最重复、最耗人力的三个环节是什么？
  - **数据层（料）**：这些环节的数据，一个小时内能拉出来吗？
  - **技术层（灶）**：有没有人能评估"AI供应商的方案靠不靠谱"？
  - **组织层（胃口）**：除了你，谁会支持AI项目？谁会暗中抵制？
- 演讲备注："这四个问题，今晚回去就能问。不需要请咨询公司。但你问完之后会得到一个答案——答案本身可能让你更焦虑。因为你知道卡在哪了，但不知道怎么解。"
- 获客暗示：答案让人焦虑→需要有人帮你解

**【PPT 15/20】四起点×四层就绪度——重合诊断**
- 核心信息：
  - B类企业——数据层卡住（有数据但散且不准）+ 技术层弱（IT只做运维）
  - C类企业——四层都弱。但可以先从业务层切入——选对数据要求最低的场景
- 演讲备注："看到了吗？不管你是什么起点，业务层永远是第一步——先想清楚'做什么菜'。别从'搞数据'开始。这是我见过最多的AI项目失败姿势——花半年做数据中台，然后发现没想清楚用来做什么。"

**【PPT 16/20】行业对标——别人在什么位置**
- 核心信息：
  - 制造业：三一(A)、海尔(A)、冰山松洋(B)、联合轴承(B)
  - 零售：沃尔玛(A)、京东(A)、美宜佳(C)、茶颜悦色(C)
- 演讲备注："看看你在同行里什么位置。如果你的对手是A，而你还在B——他要超你，不是靠更好的产品，是靠更快的AI化。"


**【PPT 17/20】互动——"三个问题，找出你的问题"**
- 类型：自测+小组讨论（同桌两人互相问）
- 三个问题（投影在屏幕上）：
  1. 你最核心的经营数据，能不能在1小时内拉出来？
  2. 你公司有没有一个人能说清楚"我们用了什么系统、每个系统里有什么数据"？
  3. 如果要推一个AI项目，除了你之外，谁会支持？谁会抵触？
- 讨论时间：3分钟
- 引导提示："跟你旁边的人互相问——不是让你回答我，是让你回答自己。如果两个都是No——你知道自己在C。一个有、一个没有——你在B。"
- 获客设计：互相问完之后，老板心里已经清楚了。但清楚之后的问题更大——"那我该怎么办？"


### Part 4：收尾与桥接（5min）

**【PPT 18/20】金句页**
- 金句："AI落地，大多数人不是在技术上失败的——而是在'以为自己准备好了'的错觉里失败的。"

**【PPT 19/20】桥接**
- 核心信息："你现在知道自己在A/B/C/D了。也知道可能卡在数据层、技术层、还是组织层。接下来的问题不是'能不能做AI'——是'怎么走'。不同的起点，不同的走法。A类可以加速跑，B类要先理数据还是边做边治，C类从小切口开始。这就是下一个模块。"
- 演讲备注："如果你现在心里在想：'我是B类，数据不准，IT只会修电脑'——那你等下一定要认真听M3。因为B类的两条路——B1先治后战还是B2边战边治——选错了可能就是几百万的代价。"

**【PPT 20/20】课后钩子**
- 核心信息："休息十分钟。如果你刚才自测完发现自己既像B又像C——来找我聊聊。"
- 获客设计：精准瞄准B/C边界人群——最需要咨询帮助的群体

## I-实施层：讲师支持

**最容易翻车的地方**：
- Part 1互动举手——如果没人举手（企业老板可能不好意思）。**应对**：讲师先举手，"我先猜——你们大多数是B。举手我看看我说对了没有？"（把诊断变成验证你的判断，降低心理门槛）
- Part 2数据不准——这个环节会制造焦虑，但不要让焦虑变成"绝望"。**应对**：在讲完数据不准之后马上接"所以B类的第一步是搞清楚数据情况——这件事你一个人推得动，不需要请AI团队"

**常见追问预判**：
1. "我觉得我们既像B又像C，怎么判断？"→"看数据。数据在系统里但不准→B。数据在Excel和微信里→C。"
2. "这些案例都是大公司，我们不是。"→"冰山松洋、联合轴承都是中型企业。你回头看这些案例的详细数据——不是A类专属。"
3. "组织层的问题怎么解？"→"这是M5要专门讲的。但我先给你一句话——不换思想就换人。"

**时间弹性方案**：
- 超时：砍Part 3的行业对标，保留四层速查4个问题
- 富余：在Part 2加一个故事——讲一个老板发现ERP库存和实际对不上的真实经历

## E-评估层：效果检验

- **下课后的检验问题**："你能不能用2×2矩阵，画出你的企业在哪个位置？"（记不住=A/B/C/D不够简单）
- **最关键指标**：休息时间有多少B类老板来问"我该选B1还是B2"——这既是获客信号，也是M3的预习需求
