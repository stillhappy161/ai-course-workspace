# 《从0到1搭建企业AI能力》课程结构

> ⚠️ 这是一个**活的文档**。框架不锁定，随知识库积累和我们之间的对话不断演化。每次调整后，智能体输出调整结果。

## 课程的真实定位

这是一门**获客课**，不是教育产品。

**表面目标**（对学员说的）：帮老板建立AI决策框架，能判断、能决策、能沟通。

**深层目标**（真正的商业目的）：
1. **利用AI焦虑**：老板对AI焦虑是真实的。课程放大焦虑的合理性，然后给清晰出路
2. **建立信任**：课程是咨询能力的demo——每个框架、诊断、案例都在证明专业判断力
3. **获取咨询客户**：老板上完课的感受不是"我学了AI"，而是"这件事我需要这个人帮我"

**内容设计检验标准**：
- 每个模块结束时，老板是否产生了"我需要帮助"的感觉？
- 框架是否简单到能记住，又深刻到让人觉得你不简单？
- 案例是否既展示了成功路径，又暗示了"自己做容易踩坑"？
- 互动是否让老板暴露了自己的问题，从而产生咨询需求？

## 模块详细大纲

### M1 认知篇：AI到底是什么（对企业而言）| 50min

**目标**：破除AI迷信，建立清晰、准确、不焦虑的AI认知。让老板知道：AI不是什么神秘力量，是一堆已经能用的工具。同时建立你的专业可信度——你不是在卖技术，是在帮老板理清决策框架。

**核心信息**：

**Part 1：开场破冰（5min）**
- 电气化革命类比——不是换发动机，是重新设计流水线
- 一句话定调：这门课不讲技术，讲决策

**Part 2：AI能力边界——"读写问答找调"（15min）**
- AI现在就能干的六件事，用五个字记：读写问答找调
  - 读（理解文档）、写（生成内容）、问（对话问答）、找（模式识别）、调（Agent调度执行）、码（写代码做分析）
- AI做不到的五件事：战略判断、担责任、理解暗知识、搞定组织政治、百分百准
- 快来了：Agent先成熟（一年内），机器人后成熟（三年左右）
- **核心信息**：AI已有的能力，够80%的企业用三年——别等完美再动手

**Part 3：AI技术栈四层——老板视角的决策地图（10min）**
- 第一层大模型（发动机）：不用纠结选哪个
- 第二层平台工具（工厂）：先用别人的
- 第三层应用层（产品）：你做决策的地方——买什么、解决什么问题
- 第四层企业集成（管道）：有坑，别自己干
- **核心信息**：第三层是你的主战场，其他三层知道即可

**Part 4：AI的三层价值——从降本到估值（10min）**
- 供应商谈第一层：降本增效
- 好顾问谈第二层：利润表结构改善（增收、降本、避险）
- 顶级顾问谈第三层：估值逻辑重构（数据资产、AI能力成为资本故事）
- **核心信息**：AI不是IT项目，是CEO项目——它的价值最终要体现在战略落地、利润表和估值上

**关键类比**：
- 电气化革命（开场）——不是换发动机，是重新设计流水线
- 海湾战争态势感知——AI第一层价值是"看清楚"

**互动**：开场自测——"你觉得你的企业AI化程度打几分？"（全场举手，1-5分）

**产出物**：一页"企业AI能力全景图"（读写问答找调六能力 + 技术栈四层 + 三层价值）
- **关联框架**：[[AI能力边界]] [[AI技术栈四层模型]] [[CEO三层关切]]

### M2 评估篇：我的企业在哪个起点 | 55min

**目标**：老板用2分钟自测，准确判断自己属于A/B/C/D哪一类。这是整个课程最关键的一步——起点判断错了，后面的路径全错。

**核心信息**：

**Part 1：一张2×2矩阵（10min）**
- 两个维度：数据基础（好/差）× 组织/技术能力（强/弱）
- 四个格子：
  - 🟢 A 子弹上膛：数据好+能力强——三一、海尔、京东
  - 🟡 B 有矿不会挖：数据有但散+能力弱——**大多数年营收5-30亿的传统企业在这里。课程核心受众。**
  - 🟡 C 两手空空：数据差+能力弱——中小企业，数据在Excel和微信里
  - 🔴 D 有枪没子弹：数据差+能力强——空降CTO、科技创业公司、集团"数字化特区"
- **关键洞察**：绝大多数中国企业是B或C，但市面上大多数AI课程讲的是A的故事。本课程的重点放在B和C。

**Part 2：数据质量陷阱（10min）——B类企业的隐藏病灶**
- "有数据"不等于"数据能用"
- 三个典型场景：
  - ERP里库存数据和实际仓库对不上（数据不准）
  - CRM里客户信息三个月没更新（数据过期）
  - 销售数据在CRM、生产数据在MES、财务数据在用友——三个系统从不说话（数据孤岛）
- **数据不准比没数据更危险**：没数据你知道自己不知道；数据不准你以为自己知道但其实不知道——基于错误数据做AI决策，后果比不做AI更糟
- B类企业的第一步不是"上AI"，是"搞清楚自己有哪些数据、数据准不准"

**Part 3：四层就绪度速查（15min）**
- 在确定起点后，用四层模型做深度自查：数据层 → 技术层 → 组织层 → 业务层
- 每层一个判断问题（老板能直接回答，不需要技术背景）：
  - 数据层：你最核心的经营数据，能不能在1小时内拉出来？
  - 技术层：你公司有没有一个人，能说清楚"我们用什么系统、每个系统里有什么数据"？
  - 组织层：如果要推一个AI项目，除了你之外还有谁会推动？谁会抵触？
  - 业务层：你业务中最重复、最耗人力的三个环节是什么？

**Part 4：行业对标（10min）**
- 制造业：三一（A）、海尔（A）、某中型汽配厂（B）
- 零售：京东（A）、茶颜悦色（C）、鸣鸣很忙（C）
- 老板看一眼就知道"我在哪个梯队"

**互动（10min）**：现场自测——发一张A4纸，一面是2×2矩阵，一面是四层速查4个问题。5分钟填完，举手看分布——现场立刻能看到A/B/C/D的比例。

**产出物**：AI起点自诊表（2×2矩阵 + 4个判断问题 + 行业对标图）

### M3 路径篇：四条路，怎么走 | 65min

**目标**：每个起点给出清晰的落地路径。**重点展开B类（核心受众）**。老板下课后知道自己下一步具体该做什么。

**核心信息**：

**Part 1：四类起点 × 四条路径总览（10min）**
- 一张总图：A路径（加速跑）/ B路径（先理后战或边战边治）/ C路径（轻骑突进）/ D路径（攒子弹或借子弹）
- 核心原则：**不管你从哪个起点出发，第一阶段都只做一件事——选一个小场景，打穿、见效果。** 区别只在于打穿之前要做什么准备。

**Part 2：B类路径——课程核心（20min）**
- **B类画像重申**：有系统（ERP/CRM/MES都上了），但数据散落、不准、不互通。IT部门有但做不了AI。年营收5-30亿的制造业/零售/物流企业。
- **B类最大的坑**：想一步到位——"我要做一个AI驱动的智能制造平台"→ 两年后还在做数据治理，老板没耐心了
- **两条子路径**：
  - **B1 先治后战**（适合：数据问题严重但场景不紧急）：
    ① 花3-6个月做数据盘点+打通（不是"完美治理"，是"够用就行"）
    ② 选第一个AI场景（建议从对数据质量要求最低的场景入手——大模型文本应用如合同审核、客服知识库，比结构化数据预测更容易启动）
    ③ 见效果 → 扩展
    - 案例：某中型制造企业，先花4个月打通ERP和MES数据，再做质检AI，良品率提升3%
  - **B2 边战边治**（适合：老板等不及，需要一个快赢）：
    ① 同时启动两条线——一条线做一个轻量AI场景（用大模型，不依赖完美数据），另一条线做数据治理
    ② 轻量场景2-3个月见效果，给数据治理争取时间和预算
    ③ 数据治理到位后，再上更重的AI场景
    - 案例：某零售连锁，一边用大模型做客服（不需要完美数据），一边花6个月做数据中台
- **B类决策指南**：选B1还是B2？看三个信号——①老板的耐心周期 ②有没有一个"不依赖数据"的快赢场景 ③IT团队有没有能力同时推进两条线

**Part 3：A类路径（5min）**
- 最大风险不是做不成，是铺太开——资源分散，哪个都没打透
- 路径：选1-2个场景集中打穿 → 建AI中台 → 全链条铺开
- 案例：三一"无人矿卡单点 → 700+场景"

**Part 4：C类路径（10min）**
- 最大风险：被忽悠买一堆用不上的系统和工具
- 路径：选一个最小最痛点 → 用现成SaaS/API（不需要自研）→ 跑通 → 再决定下一步
- **特别注意：C类企业里数据不准的那批**——先确认"你有哪些数据、数据准不准"，再决定第一个场景。如果数据完全不准，第一个场景应该是对数据依赖最低的（如用大模型做对外宣传材料的生成），而不是需要精准数据的（如销量预测）
- 案例：茶颜悦色（AI面试+门店决策，轻量切入）、鸣鸣很忙（只做巡店，打透）

**Part 5：D类路径（5min）**
- 最大风险：团队强但"无米之炊"，工程师很牛但没数据可用
- D1 攒数据：从零建数据采集体系（适合制造业、IoT）
- D2 借数据：先用外部模型和公开数据做出东西，同时建自有数据（适合对自有数据依赖不高的场景）
- D类的特殊优势：一旦数据建起来，跑得比B和C都快

**Part 6：四条路径的共同铁律（10min）**
- 第一条：第一阶段只做一个场景。贪多必死。
- 第二条：第一个场景必须"小而痛"——小到能3个月看到结果，痛到做成了有人拍桌子叫好
- 第三条：第一个场景做成了，第二件事不是做更多场景，是**把成功方法固化**——建团队、建流程、建评估标准
- 第四条：AI项目最大的失败不是技术不行，是老板不看了——保持关注、保持推动

**互动**：沙盘推演——发一个案例卡（某中型制造企业：年营收8亿，有ERP和MES但数据不对齐，IT部门3个人只做运维，老板说想上AI但不知道从哪开始）。分组讨论10分钟：判断它是A/B/C/D？选B1还是B2？第一个场景选什么？为什么？然后各组报告，讲师点评。

**产出物**：AI落地路线图一页纸模板（起点类型 → 路径选择 → 第一步场景 → 3个月里程碑 → 需要的资源）

### M4 投入篇：花多少钱、怎么花 | 40min

**目标**：老板最关心的问题——要花多少钱、钱花在哪、怎么算值不值

**核心信息**：
- AI项目费用结构：基础设施 + 软件/平台 + 数据 + 人才 + 变革管理
- 不同规模企业的投入量级（小企业几十万→大企业上千万）
- ROI的四种计算方式：降本、增收、提质、避险
- "不投AI的代价"——竞争对手在做什么
- 自建 vs 采购 vs 合作的决策树

**关键数据**：各行业AI ROI基准数据、投入产出比的真实案例

**互动**：全场投票——"你觉得AI投入最大的坑是什么？"（选项：技术不成熟 / 数据不够 / 团队跟不上 / 算不清ROI / 老板自己没想清楚）

**产出物**：AI投入决策框架图 + 预算评估模板

---

### M5 组织篇：不是CTO的事，是CEO的事 | 40min

**目标**：老板认识到AI落地最大的障碍不是技术，是组织和人

**核心信息**：
- 为什么AI项目70%的失败原因在组织而非技术
- 老板在AI化过程中的角色：变革推动者（不是技术拍板者）
- 如何设置AI负责人（不一定是CTO，可能是CDO/CAIO）
- 如何管理团队对AI的恐惧和抵触
- 案例：同样投入，组织做的好 vs 不好的企业差距有多大

**关键故事**：AI项目的翻车故事（选1-2个，要有戏剧性）

**互动**：角色扮演——"如果你的生产总监说'AI不靠谱，还是老办法好'，你怎么回应？"

**产出物**：AI变革领导力自查清单

---

### M6 行动篇：明天就可以做的三件事 | 30min

**目标**：不要让学员兴奋完回去就忘了。给三个可落地的行动。

**核心信息**：
- 行动一：做一次数据盘点（不需要技术团队，老板自己就能推动）
- 行动二：选一个"小而痛"的场景做POC
- 行动三：找一个"AI翻译官"（能帮你把商业问题翻译成技术需求的人）
- 中长期路线图工具：30-60-90天计划模板

**收尾金句**：回到电气化类比——"20世纪初，那些说'电太贵、电不稳定、还是蒸汽靠谱'的工厂主，后来去哪了？"

**互动**：课后作业——"写下你的30天承诺：接下来30天，为AI落地做哪一件事？"

**产出物**：30-60-90天AI行动计划模板
