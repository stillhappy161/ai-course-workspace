# M5 组织篇：不是CTO的事，是CEO的事 | 40min

## A-分析层

- **学员状态**：知道路径（M3）、知道花多少钱（M4）。但心里隐约知道——自己公司最大的问题不是技术也不是钱，是人
- **核心痛点**：中层抵触、IT部门跟不上、团队焦虑、老板自己不知道怎么推动。ERP时代抵触的是流程变了（5/10），AI时代抵触的是饭碗没了（9/10）
- **信任建立机会**：你是全场唯一敢说"不换思想就换人"的人——这句话从你嘴里说出来有份量，因为你做过管理咨询，你知道组织变革的代价
- **获客触发设计**：老板听完知道必须自己挂帅——但"怎么跟中层谈"、"怎么设AI负责人"、"怎么管800个Agent"——这些他需要有人帮

## D-开发层

### Part 1：开场——两个故事，一个问题（8min）

**【PPT 1/14】过渡页**
- 标题：你花钱买了AI。但你的企业接得住吗？

**【PPT 2/14】钩子——"70%的AI失败原因在组织"**
- 核心信息：不是技术不行，是人不愿意。ERP时代抵触的是"工作方式变了"——不舒服，但饭碗还在。AI时代抵触的是"饭碗还在不在"——完全不同的量级
- 演讲备注："各位，我问你们一个问题——你们公司里，谁最反对上AI？不用说出来。心里有答案就行。我敢打赌——不是最不懂AI的那个人。是最怕AI的那个人。"

**【PPT 3/14】正面案例——Block裁掉40%的人，股价涨了24%**
- 内容：Jack Dorsey判断"AI能做的事不需要人做"，在业绩强劲时将公司从10000人裁至6000人。市场奖励了这个判断——盘后涨24%。引入DRI制（Directly Responsible Individual）——从人的等级制转向问题所有权制，每个DRI在90天周期内拥有跨职能问题的完全所有权。客户问题处理周期从两周压缩到当日
- 演讲备注："你们注意到没有——他不是'不行了才裁'。他是'业绩最好的时候裁'。因为Dorsey的判断是：AI能做的事让人做，是对人的浪费，也是对股东的背叛。"

**【PPT 4/14】反面案例——钉钉ONE，AI产品不到一年就死了**
- 内容：CEO亲自挂帅的"一号工程"，巅峰DAU 300万。失败原因全在组织——目标太多（既要给员工减负、又要做AI换代、还要商业化），定位摇摆4次，AI自动标记已读做成"已读恐怖主义"，把老板当用户、普通员工被忽视。产品经理万字离职长文收尾
- 演讲备注："钉钉ONE死不是因为技术不行。它是AI'组织型失败'的教科书。你们觉得Dorsey和钉钉的区别在哪？Dorsey的AI变革是'减法'——砍人、砍层级、砍流程。钉钉是'加法'——加功能、加目标、加发心。组织变革——加法容易死，减法才能活。"

### Part 2：AI组织变革三层破局法（15min）

**【PPT 5/14】过渡——"怎么破？三层"**

**【PPT 6/14】第一层——老板的决心是唯一破局点**
- 核心信息：AI变革不能授权给CTO或HR。必须老板亲自挂帅。只有老板能打破部门墙（"把你的数据交出来"），只有老板能定"不换思想就换人"，只有老板能把AI从"IT项目"升格为"公司战略"
- 演讲备注："我问你们——如果你的CTO去找你的销售总监说'把客户数据贡献出来做AI'，销售总监会答应吗？不会。但如果你去说——'这是公司战略，不是商量'——他会交。同一个诉求，不同的人说，结果完全不同。"
- 金句："AI落地的第一件事，不是买工具——是CEO自己先成为AI的'重度用户'。"

**【PPT 7/14】第二层——用一场小胜利教育组织**
- 核心信息：第一个AI场景不只验证技术——更验证"AI不是来替代你的"。选"帮老员工干最烦的活"而非"替代老员工"。人或团队需要迅速的正反馈来激励——往往一个小的成果就能彻底改变氛围
- 演讲备注："我问你们——如果你让AI帮销售自动整理客户拜访记录，而不是替代销售做客户判断——销售会用吗？会。因为那个活他本来就不想干。AI替他干了，他就能花更多时间在客户身上。这就是小胜利——不需要大张旗鼓的组织变革，就是找一件'没人想干的活'让AI干，大家会觉得'哦，AI是帮我的'。"

**【PPT 8/14】第三层——重新定义角色，而非裁撤角色**
- 核心信息：
  - 中层：从"信息传递者"→"业务设计者"（AI处理信息，人设计规则）
  - IT部门：从"运维支撑"→"AI能力建设者"（修电脑的人变成搭AI平台的人）
  - 一线员工：从"执行者"→"AI训练师"（二十年的手感不是没用了，是变成了训练AI的养料）
- 演讲备注："你不是在裁人——你是在把人升级。不愿升级的，才需要离开。'不换思想就换人'——这句话的重点不是'换人'，是'换思想'。先给每个人机会证明自己能升级。"
- 获客暗示：角色重定义需要有人设计——这不是老板一个人能做的事

**【PPT 9/14】组织竞争力公式——为什么光有AI工具不够**
- 核心信息：**组织竞争力 = 人才密度 × AI杠杆 ÷ 组织摩擦**。分子放大但分母不动，净效果打折。很多老板拼命提升分子（买AI工具），却没动分母（审批流程、部门壁垒、信息衰减）——结果个人速度提升后，系统性堵塞更明显
- 演讲备注："你花1000万买了AI工具——这是放大分子。但你的审批流程还是7天——分母没动。等于白花钱。'先减摩擦，再加杠杆'——这个顺序不能变。"

### Part 3：组织变革怎么落地——四种涌现路径（10min）

**【PPT 10/14】过渡——"AI变革不是CEO设计出来的"**

**【PPT 11/14】四种涌现路径**
- 核心信息：四种路径可共存于同一组织——
  - **自下而上自发涌现**：超级个体互相发现、自发聚合。案例：CodeBuddy 20+领域Owner全是普通IC
  - **组织筛选培育型**：筛选→脱产训练→放回→带动全员。案例：安克创新"火箭班"、美图创新小组（10人+千万预算+分红）
  - **氛围营造型**：持续展示"别人怎么用AI做成事"。案例：腾讯AI午餐会（86%成员每天用AI，人均3-5个产品）
  - **创始人驱动型**：创始人亲自成为超级个体。案例：flomo少楠每天3小时Claude Code
- 演讲备注："这四种路径的共同点——**能力必须被看见**。你做成了AI应用，不能被锁在你的电脑里。必须在团队面前演示、在群里分享、在周会上展示。可见性是传染的前提。"

**【PPT 12/14】五个最小启动动作——管理者明天就能做**
- 核心信息：
  1. 找出已醒来的人——不按职级，看谁改了工作流
  2. 给真实问题——不是演示题，是公司真实业务痛点
  3. 让过程公开展示——可见性是传染的前提
  4. 把有效做法固化成团队资产——技能、模板、案例
  5. 同步调整激励——用AI做出成果的人，要让他被看见、被奖励
- 演讲备注："这五个动作不需要预算、不需要审批。明天就可以做。找一个人、给一个真问题、让他展示出来——这一个循环跑通，你公司就多了一个'火种'。五个火种连起来，就是一场组织变革。"

### Part 4：收尾——你准备好了吗（5min）

**【PPT 13/14】金句+追问**
- 金句："AI时代最危险的事，不是转型失败——而是继续沿着过去成功的路往前走。"
- 追问："你的中层能被升级吗？如果不能——你准备好了说那句'不换思想就换人'吗？"
- 演讲备注（停顿三秒）："我不是在威胁你。我是在帮你问自己——因为这句话，你早晚要面对。"

**【PPT 14/14】桥接+课后钩子**
- 桥接："组织变革是最难的一步。但也是最值得的一步——因为它的回报不是'省了几个人'，是'你的公司变成了AI时代能活下来的那种公司'。下一个模块，我们聊聊明天就可以做的事。"
- 获客钩子："组织变革这件事——你一个人推不动的时候，可以找我聊聊。我做过管理咨询，我知道怎么帮CEO推组织变革。"

## I-实施层：讲师支持

**最容易翻车的地方**：
- "不换思想就换人"——这句话可能让部分老板不舒服。**应对**：先讲"重新定义角色"——给每个人出路。再说"不愿升级的才需要离开"——这不是威胁，是给选择的
- Block裁40%——可能引发恐慌。**应对**：紧接着说"他们不是裁了就不管了——引入了DRI制，用新制度替代旧等级。裁人是手段，制度重构才是目的"

**常见追问预判**：
1. "我的销售总监从创业就跟着我，我不可能换他。怎么办？"→"不需要换。需要升级。先让他用AI做一个他日常最烦的事——比如自动整理拜访记录。他试过之后态度会变。"
2. "IT部门怎么升级？他们只会修电脑。"→"送一个人去学Dify——一周就能上手。不是让他们变成AI专家，是让他们变成'AI翻译官'——能帮业务部门把需求翻译成AI供应商能听懂的话。"
3. "你说四种路径——我该选哪种？"→"四种并行。你亲自成为超级个体（创始人驱动），同时找出公司里已经'醒来'的人（自下而上），送一两个人去培训（组织培育），每周做一次分享（氛围营造）。四件事同时做。"

**时间弹性方案**：
- 超时：砍Part 3四种路径的细节，只留五个最小启动动作
- 富余：加互动——"你觉得你们公司最抵触AI的是哪个部门？"
